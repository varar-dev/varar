import { Tabs, TabItem } from '@astrojs/starlight/components';

A *stimulus* drives the software: it arranges the state an example starts from
and acts on it. It is one of Varar's two kinds of step functions. The other is
[sensors](/reference/sensors/), the read-only observations. 

The names are a hardware analogy: you put a stimulus into the system, and you read its response
with sensors.

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```ts
    const { stimulus, sensor } = steps(() => ({ total: 0 }))

    stimulus('I add {int}', (state, n) => ({ total: state.total + n }))
    sensor('the total is {int}', (state) => state.total)
    ```
  </TabItem>
  <TabItem label="Java">
    ```java
    // inside defineSteps(Registrar registrar)
    record Ctx(int total) implements State {}

    StateBinder<Ctx> s = registrar.steps(() -> new Ctx(0));

    s.stimulus("I add {int}", (Ctx ctx, Integer n) -> new Ctx(ctx.total() + n));
    s.sensor("the total is {int}", (Ctx ctx, Integer expected) -> ctx.total());
    ```
  </TabItem>
  <TabItem label="Kotlin">
    ```kotlin
    data class Ctx(val total: Int = 0)

    val steps = steps(::Ctx) {
        stimulus("I add {int}") { n: Int -> copy(total = total + n) }
        sensor("the total is {int}") { _: Int -> total }
    }
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    param, stimulus, sensor = steps(lambda: {"total": 0})


    @stimulus("I add {int}")
    def _(state, n):
        return {"total": state["total"] + n}


    @sensor("the total is {int}")
    def _(state, expected):
        return state["total"]
    ```
  </TabItem>
  <TabItem label="Ruby">
    ```ruby
    steps(total: 0) do
      stimulus('I add {int}') { |state, n| { total: state[:total] + n } }
      sensor('the total is {int}') { |state, _expected| state[:total] }
    end
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    let mut s = Steps::from_registry(r);

    s.stimulus(
        "I add {int}",
        file!(),
        line!() as usize,
        Handler::sync1(|state, n| {
            let mut m = smap(&state);
            let total = m.get("total").map(as_int).unwrap_or(0) + as_int(&n);
            m.insert("total".into(), Value::Int(total));
            Ok(Some(Value::Map(m)))
        }),
    );
    s.sensor(
        "the total is {int}",
        file!(),
        line!() as usize,
        Handler::sync1(|state, _expected| Ok(smap(&state).get("total").cloned())),
    );
    ```
  </TabItem>
</Tabs>

In your prose the stimulus covers both the *context* (arrange) and the *action*
(act) — Varar never matches keywords; see
[Test anatomy](/explanation/test-anatomy/) for why the concepts and the
mechanism are decoupled.

## Evolving state

A stimulus receives the current state as its first argument (deeply readonly)
followed by the values the expression captured. In TypeScript and Python it
evolves state by **returning a partial state object**, which Varar shallow-merges
onto the current state and re-freezes. In Java and Kotlin, where state is an
immutable record/data class, it returns the **complete new state value** —
full replacement, same principle: new value out, never mutation.

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```ts
    const { stimulus } = steps(() => ({ greeting: '', count: 0 }))

    stimulus('I greet {string}', (state, name) => ({ greeting: `Hello, ${name}!` }))
    stimulus('I add {int}', (state, n) => ({ count: state.count + n }))
    ```
  </TabItem>
  <TabItem label="Java">
    ```java
    record Ctx(String greeting, int count) implements State {}

    StateBinder<Ctx> s = registrar.steps(() -> new Ctx("", 0));

    s.stimulus("I greet {string}", (Ctx ctx, String name) -> new Ctx("Hello, " + name + "!", ctx.count()));
    s.stimulus("I add {int}", (Ctx ctx, Integer n) -> new Ctx(ctx.greeting(), ctx.count() + n));
    ```
  </TabItem>
  <TabItem label="Kotlin">
    ```kotlin
    data class Ctx(val greeting: String = "", val count: Int = 0)

    val steps = steps(::Ctx) {
        stimulus("I greet {string}") { name: String -> copy(greeting = "Hello, $name!") }
        stimulus("I add {int}") { n: Int -> copy(count = count + n) }
    }
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    param, stimulus, sensor = steps(lambda: {"greeting": "", "count": 0})


    @stimulus("I greet {string}")
    def _(state, name):
        return {"greeting": f"Hello, {name}!"}


    @stimulus("I add {int}")
    def _(state, n):
        return {"count": state["count"] + n}
    ```
  </TabItem>
  <TabItem label="Ruby">
    ```ruby
    steps(greeting: '', count: 0) do
      stimulus('I greet {string}') { |_state, name| { greeting: "Hello, #{name}!" } }
      stimulus('I add {int}') { |state, n| { count: state[:count] + n } }
    end
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    s.stimulus(
        "I greet {string}",
        file!(),
        line!() as usize,
        Handler::sync1(|state, name| {
            // Full replacement: clone state, then set just the changed key.
            let mut m = smap(&state);
            m.insert("greeting".into(), Value::from(format!("Hello, {}!", as_str(&name))));
            Ok(Some(Value::Map(m)))
        }),
    );
    s.stimulus(
        "I add {int}",
        file!(),
        line!() as usize,
        Handler::sync1(|state, n| {
            let mut m = smap(&state);
            let count = m.get("count").map(as_int).unwrap_or(0) + as_int(&n);
            m.insert("count".into(), Value::Int(count));
            Ok(Some(Value::Map(m)))
        }),
    );
    ```
  </TabItem>
</Tabs>

- **Returning nothing** leaves state unchanged — right for a stimulus whose
  side effects live entirely in the system under test. (In Java and Kotlin,
  return the received state unchanged instead.)
- **In TypeScript and Python, returning anything that isn't an object** (or
  nothing) is a `ReturnShapeError`. A stimulus never returns values for
  comparison — that's the sensor's job.
- **Mutating `state` is impossible**: it is deep-frozen at runtime (TypeScript,
  Python) or an immutable record/data class (Java, Kotlin). Evolution happens
  only by returning.
- The TypeScript/Python merge is **shallow**: a returned key replaces the
  previous top-level value wholesale; nested objects are not deep-merged.

## State is per step file, per example

`steps` declares the state its step file's examples start from. Every
example gets a fresh state from the factory, so examples never leak into each
other; steps defined in different step files never see each other's state.

### No state? Omit the factory

The factory is optional. A step file whose steps are pure — nothing to
arrange, nothing to evolve — calls `steps` bare; handlers receive an
empty state they can ignore:

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```ts
    const { stimulus, sensor } = steps()

    sensor('the square of {int} is {int}', (_state, n) => [n, n * n])
    ```
  </TabItem>
  <TabItem label="Java">
    ```java
    StateBinder<State.Empty> s = registrar.steps();

    s.sensor("the square of {int} is {int}",
            (State.Empty state, Integer n, Integer expected) -> List.of(n, n * n));
    ```
  </TabItem>
  <TabItem label="Kotlin">
    ```kotlin
    val steps = steps {
        sensor("the square of {int} is {int}") { n: Int -> listOf(n, n * n) }
    }
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    param, stimulus, sensor = steps()


    @sensor("the square of {int} is {int}")
    def _(state, n, expected):
        return [n, n * n]
    ```
  </TabItem>
  <TabItem label="Ruby">
    ```ruby
    steps do
      sensor('the square of {int} is {int}') { |_state, n| [n, n * n] }
    end
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    let mut s = Steps::from_registry(r);

    s.sensor(
        "the square of {int} is {int}",
        file!(),
        line!() as usize,
        Handler::sync2(|_state, n, _square| {
            let n = as_int(&n);
            Ok(Some(Value::List(vec![Value::Int(n), Value::Int(n * n)])))
        }),
    );
    ```
  </TabItem>
</Tabs>

## Tables and doc strings

A trailing data table or fenced code block arrives as the last handler
argument, after the captured parameters — a table as a list of rows (header row
first), a doc string as its exact text:

<Tabs syncKey="lang">
  <TabItem label="TypeScript">
    ```ts
    stimulus('these books exist:', (state, rows: ReadonlyArray<ReadonlyArray<string>>) => ({
      books: rows.slice(1).map(([title, author]) => ({ title, author })),
    }))
    ```
  </TabItem>
  <TabItem label="Java">
    ```java
    s.stimulus("these books exist:", (Ctx ctx, List<List<String>> rows) -> new Ctx(
            rows.subList(1, rows.size()).stream()
                    .map(row -> new Book(row.get(0), row.get(1)))
                    .toList()));
    ```
  </TabItem>
  <TabItem label="Kotlin">
    ```kotlin
    stimulus("these books exist:") { rows: List<List<String>> ->
        copy(books = rows.drop(1).map { (title, author) -> Book(title, author) })
    }
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    @stimulus("these books exist:")
    def _(state, rows):
        return {"books": [{"title": title, "author": author} for title, author in rows[1:]]}
    ```
  </TabItem>
  <TabItem label="Ruby">
    ```ruby
    stimulus('these books exist:') do |_state, rows|
      { books: rows.drop(1).map { |title, author| { title: title, author: author } } }
    end
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    s.stimulus(
        "these books exist:",
        file!(),
        line!() as usize,
        Handler::sync1(|state, table| {
            let Value::List(rows) = table else { panic!("expected a table") };
            let books: Vec<Value> = rows
                .iter()
                .skip(1)
                .map(|row| {
                    let Value::List(cells) = row else { panic!("expected a row") };
                    vmap(vec![
                        ("title", cells[0].clone()),
                        ("author", cells[1].clone()),
                    ])
                })
                .collect();
            let mut m = smap(&state);
            m.insert("books".into(), Value::List(books));
            Ok(Some(Value::Map(m)))
        }),
    );
    ```
  </TabItem>
</Tabs>

A stimulus *consumes* these as input. To *check* a table or doc string against
what the software produced, use a sensor — see
[Check tables and doc strings](/how-to/tables-and-doc-strings/).

## Async

A stimulus handler may be async (`async` function in TypeScript, `async def`
in Python, `suspend` in Kotlin); the runtime awaits it before the next step
runs.

## Errors

| Error              | Raised when                                                        |
| ------------------ | ------------------------------------------------------------------ |
| `ReturnShapeError` | the handler returns something that isn't a partial state object or nothing |

Any exception the handler itself throws fails the example, anchored to the
step's line in the Markdown.
