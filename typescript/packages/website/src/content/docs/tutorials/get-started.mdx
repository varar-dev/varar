In this tutorial we'll install Varar into a project, configure it, run a first
[oath](/explanation/oaths/), and then deliberately break it. By the end you will
have seen an oath pass and fail as a test on your own machine.

**Pick your preferred programming language in the language selector in the top-right corner of the page**.
Every command and code sample below follows that choice.

<Steps>

1. Install

   <LangCommand kind="install" />

   This adds Varar's author API (the package your step definitions import) and the
   adapter that runs oaths in your test framework.

2. Configure

   Varar is configured by one file, `varar.config.json` — the single source of
   truth for which Markdown files are oaths and which files bind their steps.
   Ports with a `varar` CLI scaffold it (and a working example) in one command;
   on the others you create it by hand.

   <LangConfigure />

3. A first oath

   The example oath is plain Markdown with one concrete example — `varar init`
   already wrote it for you; if your port has no CLI, create it now, at any path
   matched by your config's `docs` globs:

   ```markdown title="deep-thought.md"
   # Deep Thought

   You're really not going to like it.

   The answer to the great question of life, the universe and everything is 42.

   It was a tough assignment.
   ```

   Notice there are no keywords and no special syntax — it's ordinary Markdown
   prose. One sentence makes a checkable claim; Varar matches that phrase, and
   everything around it is just documentation for the reader.

   The steps file binds that sentence to code. A **sensor** reads the software and
   returns what it actually produced, for Varar to compare against the number in the
   Markdown:

   <Tabs syncKey="lang">
     <TabItem label="TypeScript">
       ```ts title="varar-examples/steps/deep-thought.steps.ts"
       import { steps } from '@varar/varar'

       const { sensor } = steps()

       sensor('life, the universe and everything is {int}', () => 42)
       ```
     </TabItem>
     <TabItem label="Java">
       ```java title="src/test/java/examples/DeepThoughtSteps.java"
       package examples;

       import dev.varar.State;
       import dev.varar.StepDefinitions;
       import dev.varar.Steps;

       public final class DeepThoughtSteps implements StepDefinitions<DeepThoughtSteps.Ctx> {
           record Ctx() implements State {}

           @Override
           public void register(Steps<Ctx> s) {
               s.state(Ctx::new);
               s.sensor("life, the universe and everything is {int}", (Ctx ctx, Integer answer) -> 42);
           }
       }
       ```
     </TabItem>
     <TabItem label="Kotlin">
       ```kotlin title="src/test/kotlin/examples/deep-thought.steps.kt"
       @file:JvmName("DeepThoughtSteps")

       package examples

       import dev.varar.kotlin.sensor
       import dev.varar.kotlin.steps

       val deepThoughtSteps =
           steps {
               sensor("life, the universe and everything is {int}") { _: Int -> 42 }
           }
       ```
     </TabItem>
     <TabItem label="Python">
       ```python title="varar-examples/steps/deep-thought.steps.py"
       from varar import steps

       param, stimulus, sensor = steps()


       @sensor("life, the universe and everything is {int}")
       def _(state, answer):
           return 42
       ```
     </TabItem>
     <TabItem label="Ruby">
       ```ruby title="varar-examples/steps/deep-thought.steps.rb"
       require 'varar'

       steps do
         sensor('life, the universe and everything is {int}') { 42 }
       end
       ```
     </TabItem>
     <TabItem label="Rust">
       ```rust title="src/steps/deep_thought.steps.rs"
       use varar::Steps;

       pub fn register(s: &mut Steps<()>) {
           s.sensor("life, the universe and everything is {int}", |_ctx: (), _answer: i64| Ok(42));
       }
       ```
     </TabItem>
     <TabItem label="C#">
       ```csharp title="steps/deep-thought.steps.cs"
       using Varar;
       using Varar.Core;

       namespace Examples;

       public static class DeepThoughtSteps
       {
           public static void Register(Steps s)
           {
               s.Sensor("life, the universe and everything is {int}", (state, answer) => Value.Of(42));
           }
       }
       ```
     </TabItem>
     <TabItem label="Go">
       ```go title="deep_thought.steps.go"
       package examples

       import "github.com/varar-dev/varar/go/varar"

       func Register(s *varar.Steps) {
       	s.Sensor("life, the universe and everything is {int}", func(state varar.Value, args []varar.Value) (*varar.Value, error) {
       		return varar.Ptr(varar.IntValue(42)), nil
       	})
       }
       ```
     </TabItem>
   </Tabs>

4. Run it

   <LangCommand kind="run" />

   Varar reports one example passing:

   ```
   varar-examples/deep-thought.md
     ✓ Deep Thought

   1 example, 1 passed, 0 failed
   ```

   pytest, RSpec, JUnit, cargo test, dotnet test and go test each report the
   same example passing in their own format. The Markdown file just ran as a
   test, and the software kept its word.

5. Watch it fail on purpose

   Never trust a test you haven't seen fail. Open the steps file and change the
   answer the sensor returns from `42` to `43`:

   <Tabs syncKey="lang">
     <TabItem label="TypeScript">
       ```ts
       sensor('life, the universe and everything is {int}', () => 43)
       ```
     </TabItem>
     <TabItem label="Java">
       ```java
       s.sensor("life, the universe and everything is {int}", (Ctx ctx, Integer answer) -> 43);
       ```
     </TabItem>
     <TabItem label="Kotlin">
       ```kotlin
       sensor("life, the universe and everything is {int}") { _: Int -> 43 }
       ```
     </TabItem>
     <TabItem label="Python">
       ```python
       @sensor("life, the universe and everything is {int}")
       def _(state, answer):
           return 43
       ```
     </TabItem>
     <TabItem label="Ruby">
       ```ruby
       sensor('life, the universe and everything is {int}') { 43 }
       ```
     </TabItem>
     <TabItem label="Rust">
       ```rust
       s.sensor(
           "life, the universe and everything is {int}",
           file!(),
           line!() as usize,
           Handler::sync1(|_state, _answer| Ok(Some(Value::Int(43)))),
       );
       ```
     </TabItem>
     <TabItem label="C#">
       ```csharp
       s.Sensor("life, the universe and everything is {int}", (state, answer) => Value.Of(43));
       ```
     </TabItem>
     <TabItem label="Go">
       ```go
       s.Sensor("life, the universe and everything is {int}", func(state varar.Value, args []varar.Value) (*varar.Value, error) {
       	return varar.Ptr(varar.IntValue(43)), nil
       })
       ```
     </TabItem>
   </Tabs>

   Run Varar again. The example now fails: the oath still says `42`, but the sensor
   observed `43`. The output shows both values and points at the exact `42` in the
   Markdown where the promise broke.

   Revert the change and run once more — one example, one passed. You have seen
   both sides of the contract.
</Steps>
