In this tutorial we'll install Vár into a project, scaffold a working example, run
it, and then deliberately break it. By the end you will have seen an [oath](/explanation/oaths/)
pass and fail as a test on your own machine.

**Pick your preferred programming language in the language selector in the top-right corner of the page**.
Every command and code sample below follows that choice.

<Steps>

1. Install

   <LangCommand kind="install" />

   This adds Vár's author API (the package your step definitions import) and the
   adapter that runs specs in your test framework.

2. Scaffold a first spec

   <LangCommand kind="scaffold" />

   ```
   created var.config.json
   created var-examples/deep-thought.md
   created var-examples/steps/deep-thought.steps.<ext>
   ```

   (`<ext>` is your language's step-file extension — `ts`, `py`, `rb`, and so on.)

   `var.config.json` is the single source of truth for which files are specs and
   which files bind their steps:

   ```json
   {
     "docs": { "include": ["var-examples/**/*.md"], "exclude": [] },
     "steps": ["var-examples/**/*.steps.<ext>"]
   }
   ```

   The spec itself is plain Markdown with one concrete example:

   ```markdown
   # Deep Thought

   You're really not going to like it.

   The answer to the great question of life, the universe and everything is 42.

   It was a tough assignment.
   ```

   Notice there are no keywords and no special syntax — it's ordinary Markdown
   prose. One sentence makes a checkable claim; Vár matches that phrase, and
   everything around it is just documentation for the reader.

   The steps file binds that sentence to code. A **sensor** reads the software and
   returns what it actually produced, for Vár to compare against the number in the
   Markdown:

   <Tabs syncKey="lang">
     <TabItem label="TypeScript">
       ```ts
       import { steps } from '@varar/varar'

       const { sensor } = steps()

       sensor('life, the universe and everything is {int}', () => 42)
       ```
     </TabItem>
     <TabItem label="Java">
       ```java
       package examples;

       import dev.varar.Registrar;
       import dev.varar.State;
       import dev.varar.StateBinder;
       import dev.varar.StepDefinitions;

       public final class DeepThoughtSteps implements StepDefinitions {
           record Ctx() implements State {}

           @Override
           public void defineSteps(Registrar registrar) {
               StateBinder<Ctx> s = registrar.steps(Ctx::new);
               s.sensor("life, the universe and everything is {int}", (Ctx ctx, Integer answer) -> 42);
           }
       }
       ```
     </TabItem>
     <TabItem label="Kotlin">
       ```kotlin
       @file:JvmName("DeepThoughtSteps")

       package examples

       import dev.varar.kotlin.sensor
       import dev.varar.kotlin.steps

       val deepThoughtSteps =
           steps {
               sensor("life, the universe and everything is {int}") { _: Int -> 42 }
           }
       ```
     </TabItem>
     <TabItem label="Python">
       ```python
       from var import steps

       param, stimulus, sensor = steps()


       @sensor("life, the universe and everything is {int}")
       def _(state, answer):
           return 42
       ```
     </TabItem>
     <TabItem label="Ruby">
       ```ruby
       require 'varar'

       steps do
         sensor('life, the universe and everything is {int}') { 42 }
       end
       ```
     </TabItem>
     <TabItem label="Rust">
       ```rust
       use var::{Handler, Registry, Steps, Value};

       pub fn register(r: Registry) -> Registry {
           let mut s = Steps::from_registry(r);
           s.sensor(
               "life, the universe and everything is {int}",
               file!(),
               line!() as usize,
               Handler::sync1(|_state, _answer| Ok(Some(Value::Int(42)))),
           );
           s.into_registry()
       }
       ```
     </TabItem>
   </Tabs>

   <Aside type="note" title="Java & Kotlin: two extra steps">
   The JVM ports have no `var init` yet, so create `var.config.json`, the spec, and
   the steps file by hand. Two JVM-specific details:

   - `var.config.json` lists step classes by fully-qualified name instead of a glob:
     ```json
     { "docs": { "include": ["*.md"], "exclude": [] }, "steps": ["examples.DeepThoughtSteps"] }
     ```
   - add a one-line JUnit suite so Maven or Gradle discovers the specs — `RunVarSpecsTest.java` / `.kt`:
     ```java
     @Suite @IncludeEngines("var") @SelectDirectories(".") class RunVarSpecsTest {}
     ```

   The [Java](https://github.com/oselvar/vararar-examples/tree/main/java-junit-maven) and
   [Kotlin](https://github.com/oselvar/vararar-examples/tree/main/kotlin-junit) sample
   projects are complete, runnable copies you can start from.
   </Aside>

3. Run it

   <LangCommand kind="run" />

   Vár reports one example passing:

   ```
   var-examples/deep-thought.md
     ✓ Deep Thought

   1 example, 1 passed, 0 failed
   ```

   pytest, RSpec and JUnit each report the same example passing in their own
   format. The Markdown file just ran as a test, and the software kept its word.

4. Watch it fail on purpose

   Never trust a test you haven't seen fail. Open the steps file and change the
   answer the sensor returns from `42` to `43`:

   <Tabs syncKey="lang">
     <TabItem label="TypeScript">
       ```ts
       sensor('life, the universe and everything is {int}', () => 43)
       ```
     </TabItem>
     <TabItem label="Java">
       ```java
       s.sensor("life, the universe and everything is {int}", (Ctx ctx, Integer answer) -> 43);
       ```
     </TabItem>
     <TabItem label="Kotlin">
       ```kotlin
       sensor("life, the universe and everything is {int}") { _: Int -> 43 }
       ```
     </TabItem>
     <TabItem label="Python">
       ```python
       @sensor("life, the universe and everything is {int}")
       def _(state, answer):
           return 43
       ```
     </TabItem>
     <TabItem label="Ruby">
       ```ruby
       sensor('life, the universe and everything is {int}') { 43 }
       ```
     </TabItem>
     <TabItem label="Rust">
       ```rust
       s.sensor(
           "life, the universe and everything is {int}",
           file!(),
           line!() as usize,
           Handler::sync1(|_state, _answer| Ok(Some(Value::Int(43)))),
       );
       ```
     </TabItem>
   </Tabs>

   Run Vár again. The example now fails: the spec still says `42`, but the sensor
   observed `43`. The output shows both values and points at the exact `42` in the
   Markdown where the promise broke.

   Revert the change and run once more — one example, one passed. You have seen
   both sides of the contract.
</Steps>
