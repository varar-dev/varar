# Rename bdd → Vár (var) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rename the library from `bdd`/`bdds`/`bididi` to **Vár** (`var`) across packages, files, code identifiers, and branding, keeping the methodology term "BDD" intact.

**Architecture:** A pnpm monorepo. The rename is mechanical but order-sensitive: the lowercase token `bdd` is ambiguous (a code identifier vs. the product name), and `var` is a reserved JS keyword, so identifiers use `varDoc`/`VarDoc`. The repo will not build until *all* code tokens are consistent, so the real green gate lives at the end of the code-token task; earlier tasks gate on structural greps and `pnpm install`.

**Tech Stack:** pnpm workspace · TypeScript (ESM) · vitest · biome · knip · jscpd · Astro (website). `git mv` for moves, `perl -pi` sweeps for content.

## Global Constraints

- **Keyword safety:** `var` is a reserved word. It is legal as a package name, a CLI command string, a config key, and an object key, but NEVER as a bare JS identifier (`const var`, `function f(var)`). Document-typed identifiers therefore use `VarDoc` (types) / `varDoc` (locals/params/fields).
- **Methodology "BDD" is kept verbatim.** "BDD" / "Behaviour-Driven Development" naming the *methodology* (e.g. `var — markdown-native BDD`) must survive. Only the *product* names `bdd`, `bdds`, `bididi` become Vár/var. Uppercase `BDD` is never blanket-replaced.
- **Runic / accent are branding only.** ᚢᛅᚱ and "Vár" appear in README/docs/website/`ANNOUNCEMENT.md` titles & logo. All code, package names, CLI output, and config use ASCII `var`.
- **`@oselvar` scope is kept.** Only the `bdd` segment changes.
- **Out of scope:** `packages/*/dist/` (gitignored, regenerated by build), everything under `docs/superpowers/` (historical specs/plans — left as a record), and renaming the git repo / working directory.
- **Caution — do not touch:** the UUID fragments `4b9d-bdd0-...` in `scripts/install-vscode.mjs` (the `bdd0` substring is part of a UUID, matched by none of our patterns — verify it is unchanged at the end).

## The `sweep` helper

Several tasks reuse one helper. Define it in the shell at the start of each task (it is not persisted between Bash calls):

```bash
# Apply a perl s/// expression across all tracked text files,
# excluding historical superpowers docs and the lockfile.
sweep() {
  git ls-files \
    | grep -vE '^docs/superpowers/' \
    | grep -vE '(^|/)pnpm-lock\.yaml$' \
    | grep -E '\.(ts|tsx|js|mjs|cjs|json|jsonc|md|astro|yaml|yml|css|html)$' \
    | tr '\n' '\0' \
    | xargs -0 perl -pi -e "$1"
}
```

`git ls-files` lists only tracked files (so `node_modules/` and `dist/` are already excluded) and reflects the *current* paths after each committed `git mv`.
