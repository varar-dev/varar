# Making the docs agent-friendly

The primary audience for the Varar docs is coding agents. This note records how
we serve them and — crucially — how to unblock them at the CDN, which is the one
part that cannot live in this repo.

## The two layers

Agent-friendliness splits cleanly in two:

1. **Reachability (CDN / Cloudflare).** Can an agent's HTTP client get a `200`
   at all? This is enforced by Cloudflare *before* any static asset is served
   and is configured in the Cloudflare dashboard, **not** in this repo.
2. **Consumability (this repo).** Once a request is served, is the content easy
   for a model to ingest? This is `llms.txt`, `robots.txt`, and the sitemap,
   all built from `typescript/packages/website`.

Adding files in layer 2 does **nothing** for layer 1. A permissive `robots.txt`
does not override Bot Fight Mode; `_headers` runs after the bot check. If an
agent reports a `403`, the fix is always in the Cloudflare dashboard.

## Layer 1 — fixing a 403 in Cloudflare

varar.dev is a Cloudflare Workers Static Assets deployment (see
`typescript/packages/website/wrangler.jsonc`). A `403` to a non-browser client
is almost always one of these zone settings. Check them in order:

1. **Bot Fight Mode** — Security → Bots. The free tier's blunt instrument: it
   challenges/blocks all automated, non-browser traffic, which is every coding
   agent. Turn it **off** for this zone. (On paid plans this is *Super* Bot
   Fight Mode: set "Automated traffic" to *Allow* instead of turning it off
   wholesale.)
2. **"Block AI bots" / AI Crawl Control** — Security → Bots. Cloudflare's
   one-click AI-crawler block, which is on by default for some recently
   onboarded domains. It matches by user-agent (`ClaudeBot`, `GPTBot`, …). Set
   the bots you want to *Allow*. At minimum allow **`Claude-User`** — that is
   the *user-initiated* fetcher (a person asked Claude about a page), distinct
   from `ClaudeBot` (the training crawler). You want `Claude-User` allowed even
   if you block training crawlers.
3. **A WAF custom rule to Skip** (durable belt-and-suspenders) — Security → WAF
   → Custom rules. Expression `(http.host eq "varar.dev")`, action **Skip**,
   skipping Super Bot Fight Mode and all managed rules. The whole domain is
   public docs meant to be read by machines, so a blanket skip is defensible.

There is no way to set these from `wrangler.jsonc`; they are zone settings.
Terraform (`cloudflare_bot_management`, `cloudflare_ruleset`) can codify them if
we want them in version control later, but that is out of scope for the docs
build.

To verify a fix, fetch as a non-browser client:

```sh
curl -sS -A 'Claude-User' -o /dev/null -w '%{http_code}\n' https://varar.dev/
curl -sS -A 'Claude-User' https://varar.dev/llms.txt | head
```

## Layer 2 — what this repo ships

Built from `typescript/packages/website`:

- **`/llms.txt`** and **`/llms-full.txt`** — generated at build time by the
  `starlight-llms-txt` Starlight plugin (wired in `astro.config.mjs`).
  `llms.txt` is a curated index; `llms-full.txt` is every doc page concatenated
  as clean Markdown, so an agent can ingest the whole corpus from one URL
  instead of scraping rendered HTML.
- **`/robots.txt`** (`public/robots.txt`) — allows all crawlers, names the AI
  user-agents explicitly as welcome, and advertises the sitemap. Advisory only.
- **`/sitemap-index.xml`** — generated by `@astrojs/sitemap`, referenced from
  `robots.txt`.
