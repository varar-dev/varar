# Go port — task plan

Date: 2026-07-19
Status: in progress

Implements the Go port per [ADR 0010](../../adr/0010-go-port.md),
[ADR 0011](../../adr/0011-go-test-integration.md), and the two design specs
([core](../specs/2026-07-19-go-core-port-design.md),
[runner+adapter](../specs/2026-07-19-go-runner-adapter-design.md)). Each core
stage is gated by reproducing the shared goldens byte-for-byte before the next.

## Follow-ups

- [x] **Typed step constructors** (`varar.Sensor1/2/3`, `varar.Stimulus0/1/2`),
      generic rather than godog's reflection: each slot arrives as a plain Go
      value and a sensor returns one value per slot, mirroring the TS tuple
      return. Additive — the explicit `[]Value` form stays the primitive for
      whole tables, header-bound rows, and custom types that parse to a map.
      See the handler-shape fork point in [ADR 0010](../../adr/0010-go-port.md).
- [ ] Decide whether the website docs' Go tabs should lead with the typed form
      (they currently show the explicit one).
- [ ] Consider a dedicated `SensorRow` constructor: header-bound rows compare by
      column, so `Sensor1[Value]` composes but understates the contract.
